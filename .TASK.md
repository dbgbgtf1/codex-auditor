# codex-auditor webui

1. 绑定到nginx `/ui/`路由，主体部分不需要强制/ui路由
2. 使用codex exec和ai交互
3. 设计会话绑定功能，保存打开过的session，如果需要新开会话，就保存session，如果要恢复会话就用旧session
4. 使用sqlite数据库保存信息
5. 每个会话指定提示词以及标识符（根据AGENTS的信息）
6. 每个会话显示挖掘到的漏洞，从audit目录对应标识符的目录中拿到json解析；不需要缓存；如果读取失败，则要求ai根据skill更新
7. 用gpt 5.4 mini判断当主agent（gpt 5.5）停下来时发挥来的消息，如果没有必要停止的话就发送继续，否则需要把信息上报到web ui上

## ui设计

1. 不需要密码，已由nginx验证
2. 设计暗色模式和亮色模式的配色，并把模式缓存到数据库中；亮色主题的颜色主要为类似阅读模式的纸黄色；暗色主题的颜色主要为接近黑色的深灰色
3. 边栏设置为新建会话的按钮、暗色主题的按钮以及竖向排列的会话列表
4. 右侧上半页设计为漏洞列表显示，以表来表示，其中危害程度用颜色做区分
5. 右侧下半页设计为传统的聊天样式，最下面是横条对话框，然后主要部分显示为使用的消息（右）与agent的消息（左）
6. 不要出现一开始是亮色，然后读取到是暗色才切换为暗色；先判断主题，再加载ui
7. 如果overall.json不可用，就显示暂无漏洞
8. 允许删除会话

## 代码部分

1. 可以使用arch中存在的，且稳定的库作为依赖
2. web响应用flask来构建
3. 使用python 3.14作为基准编写代码，包含类型注释
4. 增加pyproject.toml来记录依赖
5. 在交付代码时需要通过mypy、ruff的规范验证，注意不能为了通过而使用Any，尽量避免Any类型的注释
