# 标题

## 结论

置信度和安全相关性(low/medium/high)。

## 环境

目标 commit 或版本、构建类型、二进制、sanitizer/debug 事实和关键 flag。

## 复现方法

从工作区根目录运行的命令、复现判定和重复次数。

## 实际现象

crash、sanitizer 报告、错误结果、断言、状态破坏或 trace 差异。

## 预期行为

说明实际现象为什么违反源码边界条件、公开 API 契约、内存安全规则、协议状态推进规则或认证状态推进规则。

## 根因分析

源码路径、函数、分支、输入或状态来源、边界条件、状态改变、危险使用点，以及未满足的检查或安全规则。

## 影响判断

影响的版本范围和时间范围。内存破坏、拒绝服务、权限/安全边界影响、逻辑 bug、debug-only 问题或未知。
